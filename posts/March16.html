A checksum is a small piece of data derived from a large piece of data. Detecting errors is the purpose of creating a checksum. Errors can occur during transmission or storage of data.

The following is a snippet of code that accepts a string and will create a checksum for that string. We can make this code is designed to handle small strings. Larger strings would be better handled by a checksum of the data type long. It would not be space efficient to allocate a long variable to hold a number smaller than 2,147,483,647 (2^32/2). We may need a number larger than 2 billion if we are transmitting a string as large as a text book.

Each letter in the string is translated into its Ascii value. The Ascii values are added up into a checksum. Identical strings will always add up to the same checksum. This process only works one way. Many strings can yield the same checksum. When a packet is sent, the checksum is calculated and sent along with the packet as part of the header. When the packet arrives at its destination the receiver can verify the string is valid by recalculating the checksum. If the received checksum matches the calculated checksum it is very unlikely that the packet was corrupt.
