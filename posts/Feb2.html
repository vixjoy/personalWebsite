Kali Linux, Metasploitable, and Rickdiculously Easy

Posted by Victoria on January 26th, 2018

This week I made great strides in learning. I set up a great pen testing lab with Virtual Box, I learned a lot about networking. After getting Kai Linux up and running, I started poking around with Metasploitable and Rickdiculously easy virtual machines.

The biggest challenge I had this week was simulating separate machines, on the same network. The tricky part of this is the security of my personal computer. When Metasploitable is plainly exposed to the internet, anyone could hack into it. There exist paths between my personal computer and Metasploitable. For security reasons, exposing Metasploitable to the internet simply would not do. After many failed attempts to get each virtual machine a unique IP address and a common net mesh, I created a host only network.

The host only network connects the guest operating systems Kali Linux, Rickdiculously easy, and Metasploitable to each other. The network connection comes in and out via my host operating system Sierra. My host machine is configured securely. In theory, one should be barred from seeing my vulnerable virtual machines unless they obtained access to my computer first.

Commands I found extremely helpful in configuring the networks include,

  • nmap
  • nmap -A
  • ifconfig
  • ifconfig eth0
